# Endringer 2026 Q1

*Oppdater denne siden med nye funksjoner og endringer.*

## Database — Transaction Pooler (feb 2026)

- Migrert database-tilkobling fra Session Pooler (`port 5432`, `max: 1`) til **Transaction Pooler** (`port 6543`, `max: 10`).
- Ny miljøvariabel `DATABASE_MIGRATION_URL` — Session Pooler brukes for migrasjoner og Drizzle Studio, Transaction Pooler for app og worker.
- `prepare: false` satt på alle tilkoblinger (påkrevd for PgBouncer transaction mode).
- Worker oppdatert med `prepare: false`, `idle_timeout` og `connect_timeout`.
- Migrasjonsskript oppdatert til å foretrekke `DATABASE_MIGRATION_URL`.

## Ekstern dokumentinnhenting (feb 2026)

- Ny funksjon: **Be om dokumentasjon fra eksterne** via magic link.
- Intern bruker oppretter en forespørsel fra «Følg opp»-dialogen (matching-visningen) eller «Ny oppgave»-dialogen.
- Ekstern kontakt mottar e-post med en unik opplastingslenke — ingen innlogging nødvendig.
- Offentlig opplastingsside (`/d/[token]`) med drag-and-drop og flerfilopplasting.
- Opplastede filer knyttes automatisk til riktig transaksjon og oppgave.
- Intern bruker varsles i appen når dokumentasjon er mottatt.
- Lenker utløper etter 14 dager (konfigurerbart).
- Ny databasetabell `document_requests` og `document_request_files` med RLS.
