# Quality of Life — forbedringer

Denne mappen dokumenterer **småskala UX-forbedringer** som øker brukervennligheten uten å endre kjernelogikk. Formålet er todelt:

1. **Salg og presentasjon** — samlet oversikt over detaljgraden i produktet, som kan brukes i pitch til kunder og ledelse.
2. **Intern historikk** — sporbarhet for småforbedringer som ellers forsvinner i commit-loggen.

## Filformat

- Én `.md`-fil per forbedring eller tematisk gruppe
- Filnavn: `YYYY-MM-DD-kort-beskrivelse.md`
- Hvert dokument inneholder:
  - **Hva** — kort beskrivelse av endringen
  - **Hvorfor** — problemet det løser for brukeren
  - **Hvordan** — teknisk oversikt (hvilke filer/moduler)
  - **Før/etter** — visuell eller tekstlig sammenligning der det gir mening

## Bruk i pitch

Disse dokumentene kan samles til en **«Detaljene som teller»**-seksjon i salgsmateriell. Poenget er å vise at produktet er bygd med norske regnskapsbrukere i sentrum — ned til tusenskilletegn og datoformat.
