# Endringsdokumentasjon

Denne mappen inneholder **lokale notater og historikk** over endringer som gjøres i prosjektet underveis. Formålet er å ha en enkel, lesbar dokumentasjon av hva som er bygget eller endret, uten å måtte grave i git-loggen.

## Bruk

- **Én .md-fil per tema eller dato** – f.eks. `2025-02-19-import-excel.md`
- **Korte, konkrete beskrivelser** – hva ble endret, hvorfor, og evt. hvordan det brukes
- **Filnavn** – anbefalt format: `YYYY-MM-DD-kort-beskrivelse.md` slik at rekkefølgen er tydelig

Filer her er ment som **supplement** til commit-meldinger og kan referere til konkrete filer eller moduler i kodebasen.
